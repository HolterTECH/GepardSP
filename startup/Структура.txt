

ПОЛНАЯ СТРУКТУРА ПРОЕКТА - КАНБАН ДОСКА И СЕТЕВОЙ ГРАФИК
=====================================================

ОБЩЕЕ ОПИСАНИЕ:
- Проект: Система управления проектами с канбан-доской и сетевым графиком
- Стек: Python + Flask + SQLite + HTML/CSS/JavaScript
- База данных: SQLite (файл instance/app.db)

ПОДРОБНОЕ ОПИСАНИЕ КАЖДОГО ФАЙЛА:
================================

1. APP.PY - ГЛАВНЫЙ ФАЙЛ ПРИЛОЖЕНИЯ
----------------------------------
НАЗНАЧЕНИЕ: Основной файл Flask приложения, содержит всю серверную логику

ФУНКЦИОНАЛ:
- Инициализация базы данных
- Маршруты (URL endpoints) для всех страниц
- API endpoints для работы с задачами и проектами
- Обработка HTTP запросов

ОСНОВНЫЕ МАРШРУТЫ:
- GET  /                    - Главная страница (список проектов)
- GET/POST /project/create  - Создание нового проекта
- GET  /project/<id>/kanban - Канбан-доска проекта
- GET  /project/<id>/network - Сетевой график проекта

API ENDPOINTS:
- GET    /api/projects                  - Получить все проекты
- GET    /api/project/<id>/tasks        - Получить задачи проекта
- POST   /api/project/<id>/task         - Создать новую задачу
- GET    /api/task/<id>                 - Получить задачу
- PUT    /api/task/<id>                 - Обновить задачу
- POST   /api/task/<id>/status          - Обновить статус задачи
- POST   /api/task/<id>/dates           - Обновить даты задачи
- DELETE /api/task/<id>                 - Удалить задачу
- POST   /api/task/<id>/dependencies    - Установить зависимости задач

2. TEMPLATES/INDEX.HTML - ГЛАВНАЯ СТРАНИЦА
-----------------------------------------
НАЗНАЧЕНИЕ: Отображает список всех проектов

ФУНКЦИОНАЛ:
- Показывает все существующие проекты
- Кнопка "Создать проект"
- Ссылки на канбан-доску и сетевой график для каждого проекта
- Простой и понятный интерфейс

ОСОБЕННОСТИ:
- Адаптивный дизайн
- Карточки проектов
- Навигация между страницами

3. TEMPLATES/CREATE_PROJECT.HTML - СОЗДАНИЕ ПРОЕКТА
--------------------------------------------------
НАЗНАЧЕНИЕ: Форма для создания нового проекта

ФУНКЦИОНАЛ:
- Поле ввода названия проекта
- Поле ввода описания проекта
- Кнопка создания проекта
- Валидация данных

4. TEMPLATES/KANBAN.HTML - КАНБАН ДОСКА
---------------------------------------
НАЗНАЧЕНИЕ: Основной интерфейс управления задачами

ФУНКЦИОНАЛ:
- 3 колонки: "Запланировано", "В работе", "Выполнено"
- Drag & Drop перемещение задач между колонками
- Создание новых задач
- Редактирование задач (модальное окно)
- Удаление задач
- Управление датами: начало, окончание, длительность
- Автоматический расчет дат при изменении длительности

ОСОБЕННОСТИ:
- Интуитивный интерфейс перетаскивания
- Валидация дат
- Автоматические вычисления
- Модальные окна для редактирования

5. TEMPLATES/NETWORK.HTML - СЕТЕВОЙ ГРАФИК
------------------------------------------
НАЗНАЧЕНИЕ: Визуализация зависимостей между задачами

ФУНКЦИОНАЛ:
- Графическое отображение задач как узлов
- Стрелки показывают зависимости между задачами
- Цветовая кодировка статусов задач
- Редактирование зависимостей
- Расчет критического пути
- Управление отображением графа

ОСОБЕННОСТИ:
- Использует библиотеку vis.js для визуализации
- Интерактивный граф (масштабирование, перемещение)
- Модальные окна для управления зависимостями
- Автоматическое расположение узлов

БАЗА ДАННЫХ (SQLITE):
====================

ТАБЛИЦА projects:
- id (INTEGER) - уникальный идентификатор
- name (TEXT) - название проекта
- description (TEXT) - описание проекта

ТАБЛИЦА tasks:
- id (INTEGER) - уникальный идентификатор
- project_id (INTEGER) - ссылка на проект
- title (TEXT) - название задачи
- description (TEXT) - описание задачи
- status (TEXT) - статус: 'planned', 'in_progress', 'completed'
- position (INTEGER) - позиция в колонке канбана
- duration (INTEGER) - длительность в днях
- start_date (DATE) - дата начала
- end_date (DATE) - дата окончания
- dependencies (TEXT) - зависимости (список ID через запятую)

КАК ЗАПУСТИТЬ ПРОЕКТ:
====================

1. Убедитесь, что установлен Python 3.7+
2. Установите Flask: pip install flask
3. Поместите все файлы в правильную структуру папок
4. Запустите в терминале: python app.py
5. Откройте браузер: http://127.0.0.1:5000

ТЕХНИЧЕСКИЕ ОСОБЕННОСТИ:
=======================

- Frontend: Чистый JavaScript (без фреймворков)
- Backend: Flask (минималистичный фреймворк)
- База данных: SQLite (встроенная в Python)
- Визуализация: vis.js для сетевого графика
- Стили: CSS3 с Flexbox/Grid
- Адаптивный дизайн

ПРИМЕЧАНИЯ:
==========

- Проект использует только стандартные библиотеки Python
- Не требует установки дополнительных зависимостей кроме Flask
- База данных создается автоматически при первом запуске
- Все данные хранятся локально в файле SQLite
- Проект легко расширять и модифицировать
